## Final Featured Projects

The homepage featured projects are:

1. Mahoraga
2. Hybrid GenAI Transaction Categorizer
3. The Loop

This selection is considered finalized unless new evidence reveals a critical contradiction, such as a missing repo, no demo artifact, or insufficient ownership clarity.

## Selection Rationale

## 1. Mahoraga

Role in portfolio:
- Flagship technical depth.
- Primary proof of adaptive AI systems.

Why it is featured:
- Shows adaptive agent behavior, RL environment design, reward engineering, LoRA fine-tuning, quantization, API/interface work, and experimentation.
- Differentiates AJ from generic AI wrapper portfolios.
- Supports the primary identity: AI Systems Engineer.

Required framing:
- Lead with adaptive agent systems and reward design.
- Keep game/anime context secondary.
- Show environment, reward, training/evaluation, and agent behavior.

Risk:
- Can be misread as a themed game project if the page leads with surface context.

Required evidence:
- Architecture diagram.
- Reward-system visualization.
- Training timeline or metrics artifact.
- Demo GIF or video.
- Kaggle or HuggingFace link if available.
- Repo link if available.
- Limitation or failure-mode note.

## 2. Hybrid GenAI Transaction Categorizer

Role in portfolio:
- Cleanest proof of production-minded AI systems architecture.

Why it is featured:
- Shows ONNX inference, confidence routing, LLM fallback, merchant extraction, memory, prediction history, and analytics.
- Demonstrates cost/latency/accuracy tradeoff thinking.
- Makes the AI Systems Engineer positioning concrete and practical.

Required framing:
- Lead with routing architecture.
- Explain why not every request should go to an LLM.
- Show confidence thresholds, fallback behavior, merchant memory, and analytics flow.

Risk:
- Without benchmarks or routing examples, it can look like a classifier demo.

Required evidence:
- Confidence-routing diagram.
- Merchant memory visualization.
- Latency or routing comparison table.
- UI screenshots.
- Demo video or GIF.
- Example transactions showing routing decisions.
- Repo and deployment links if available.

## 3. The Loop

Role in portfolio:
- Main proof of Product Engineer identity.
- Balances the portfolio so it does not look AI-only.

Why it is featured:
- Shows full-stack product engineering: React, FastAPI, PostgreSQL, SQLAlchemy, OAuth, JWT, WebSockets, recommendations, and social workflows.
- Proves AJ can build user-facing systems, not only AI experiments.

Required framing:
- Lead with the student coordination problem and product workflow.
- Show database, auth, WebSocket, recommendation, and deployment-facing architecture.
- Keep product value visible before stack details.

Risk:
- If visually undersold or moved to secondary, the portfolio becomes too AI-heavy.

Required evidence:
- Product screenshots.
- Database diagram.
- WebSocket architecture diagram.
- Auth flow diagram.
- User workflow walkthrough.
- Repo and deployment links if available.

## Project Hierarchy Rules

1. Featured projects get the largest visual treatment and full case studies.
2. Secondary projects get compact cards or shorter case studies.
3. Lab projects get compact entries.
4. Timeline-only projects remain small progression markers.
5. Weak projects must never compete visually with Mahoraga, Hybrid Categorizer, or The Loop.
6. Every project must include status specificity and evidence availability.
7. No project gets a demo/repo/deployment label unless the link is verified.
